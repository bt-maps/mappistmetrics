a lean, mean, computing machine for realtime geospatial analytics


The mappist-metrics server is a set of scripts and configuration files to run a powerful application server for geofencing, counters, and timeseries data. It speaks HTTP, WebSockets, Server-Sent Events, RESP and it's FAST!

It's perfect for-

  • Building fast geospatial and timeseries APIs with Redis and Tile38
  • Realtime counting engines and leaderboards
  • Timeseries storage and aggregation servers
  • Publish-subscribe servers with notifications over websocket and SSE

Components

OpenResty
Dynamic web platform based on NGINX and LuaJIT
NCHAN
Scalable, flexible pub/sub server for the modern web
Tile38
Ultra fast geospatial database & geofencing server
Redis
In-memory data structure server
Lua
A powerful, dynamic and light-weight programming language

For Lua scripting, I recommend ZeroBrane Studio, a lightweight Lua IDE with code completion, syntax highlighting, live coding, code analyzer, and debugging support Download

Basic set-up

  1. configure and set your geofences and notifications channels
  2. configure your counters with the json-based config
  3. ingest your moving object data
  4. retrieve your geospatial data, notifications, counters, raw and aggregated timeseries using the API and channels
